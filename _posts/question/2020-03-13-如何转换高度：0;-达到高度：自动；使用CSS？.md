          <div class="discuss-comment"><p><font style="vertical-align: inherit;"><font style="vertical-align: inherit;">如果提供的硬编码最大高度值不大于实际高度，则杰克（Jake）的最大高度解决方案可以很好地工作（因为否则会有不希望的延迟和计时问题）。</font><font style="vertical-align: inherit;">另一方面，如果硬编码值偶然不大于实际高度，则该元素将不会完全打开。</font></font></p>

<p><font style="vertical-align: inherit;"><font style="vertical-align: inherit;">以下仅CSS解决方案还需要硬编码的大小，该大小应大于大多数实际大小。</font><font style="vertical-align: inherit;">但是，如果实际大小在某些情况下大于硬编码大小，则此解决方案也适用。</font><font style="vertical-align: inherit;">在那种情况下，过渡可能会跳一点，但绝不会留下部分可见的元素。</font><font style="vertical-align: inherit;">因此，该解决方案还可以用于未知内容，例如来自数据库的未知内容，您只知道其中的内容通常不大于x像素，但是也有例外。</font></font></p>

<p><font style="vertical-align: inherit;"><font style="vertical-align: inherit;">想法是对底边距使用负值（对于稍有不同的动画，对边距顶部使用负值），然后将内容元素放入带有overflow：hidden的中间元素中。</font><font style="vertical-align: inherit;">内容元素的负边距会降低中间元素的高度。</font></font></p>

<p><font style="vertical-align: inherit;"><font style="vertical-align: inherit;">下面的代码使用从-150px到0px的margin-bottom过渡。</font><font style="vertical-align: inherit;">只要content元素不高于150px，这单独就可以正常工作。</font><font style="vertical-align: inherit;">另外，它对中间元素的最大高度使用了从0px到100％的过渡。</font><font style="vertical-align: inherit;">如果content元素高于150px，则最终将隐藏中间元素。</font><font style="vertical-align: inherit;">对于最大高度，过渡仅用于在关闭时将其应用延迟一秒，而不是获得平滑的视觉效果（因此过渡范围可以从0px到100％）。</font></font></p>

<p><font style="vertical-align: inherit;"><font style="vertical-align: inherit;">CSS：</font></font></p>

<pre><code>.content {<font></font>
  transition: margin-bottom 1s ease-in;<font></font>
  margin-bottom: -150px;<font></font>
}<font></font>
.outer:hover .middle .content {<font></font>
  transition: margin-bottom 1s ease-out;<font></font>
  margin-bottom: 0px<font></font>
}<font></font>
.middle {<font></font>
  overflow: hidden;<font></font>
  transition: max-height .1s ease 1s;<font></font>
  max-height: 0px<font></font>
}<font></font>
.outer:hover .middle {<font></font>
  transition: max-height .1s ease 0s;<font></font>
  max-height: 100%<font></font>
}<font></font>
</code></pre>

<p><font style="vertical-align: inherit;"><font style="vertical-align: inherit;">HTML：</font></font></p>

<pre><code>&lt;div class="outer"&gt;<font></font>
  &lt;div class="middle"&gt;<font></font>
    &lt;div class="content"&gt;<font></font>
      Sample Text<font></font>
      &lt;br&gt; Sample Text<font></font>
      &lt;br&gt; Sample Text<font></font>
      &lt;div style="height:150px"&gt;Sample Test of height 150px&lt;/div&gt;<font></font>
      Sample Text<font></font>
    &lt;/div&gt;<font></font>
  &lt;/div&gt;<font></font>
  Hover Here<font></font>
&lt;/div&gt;<font></font>
</code></pre>

<p><font style="vertical-align: inherit;"><font style="vertical-align: inherit;">边距下限值应为负，并尽可能接近内容元素的实际高度。</font><font style="vertical-align: inherit;">如果它的（绝对值）较大，则存在与最大高度解决方案类似的延迟和时序问题，但是，只要硬编码的大小不比实际大小大很多，就可以限制它们。</font><font style="vertical-align: inherit;">如果边缘下限的绝对值小于实际高度，则过渡会略微跳跃。</font><font style="vertical-align: inherit;">在过渡之后的任何情况下，内容元素都将完全显示或完全删除。</font></font></p>

<p><font style="vertical-align: inherit;"><font style="vertical-align: inherit;">这样，对于</font></font><code>height: 0</code><font style="vertical-align: inherit;"><font style="vertical-align: inherit;">（有效完全折叠）的元素，其“正常”或“完整”高度仍可通过其</font></font><code>scrollHeight</code><font style="vertical-align: inherit;"><font style="vertical-align: inherit;">值（不变地为</font></font><em><font style="vertical-align: inherit;"><font style="vertical-align: inherit;">像素</font></font></em><font style="vertical-align: inherit;"><font style="vertical-align: inherit;">长度）</font><font style="vertical-align: inherit;">轻松获得</font><font style="vertical-align: inherit;">。</font></font></p>

<p><font style="vertical-align: inherit;"><font style="vertical-align: inherit;">对于这样的元素，假设它已经设置了转换，例如（</font></font><code>ul</code><font style="vertical-align: inherit;"><font style="vertical-align: inherit;">按照原始问题使用）：</font></font></p>

<pre><code>ul {<font></font>
    height: 0;<font></font>
    transition: height 1s; /* An example transition. */<font></font>
}<font></font>
</code></pre>

<p><font style="vertical-align: inherit;"><font style="vertical-align: inherit;">我们可以仅使用CSS触发所需的动画高度“扩展”，如下所示（此处假设</font></font><code>ul</code><font style="vertical-align: inherit;"><font style="vertical-align: inherit;">变量是指列表）：</font></font></p>

<pre><code>ul.style.height = ul.scrollHeight + "px";
</code></pre>

<p><font style="vertical-align: inherit;"><font style="vertical-align: inherit;">而已。</font><font style="vertical-align: inherit;">如果您需要折叠列表，则可以使用以下两个语句之一：</font></font></p>

<pre><code>ul.style.height = "0";<font></font>
ul.style.removeProperty("height");<font></font>
</code></pre>

<p><font style="vertical-align: inherit;"><font style="vertical-align: inherit;">我的特定用例围绕着动画列表，这些列表的长度未知且通常很长，因此我不适应使用任意的“足够大” </font></font><code>height</code><font style="vertical-align: inherit;"><font style="vertical-align: inherit;">或</font></font><code>max-height</code><font style="vertical-align: inherit;"><font style="vertical-align: inherit;">规范，并冒着截断的内容或突然需要滚动的内容的风险（</font></font><code>overflow: auto</code><font style="vertical-align: inherit;"><font style="vertical-align: inherit;">例如，</font><font style="vertical-align: inherit;">如果</font><font style="vertical-align: inherit;">） 。</font><font style="vertical-align: inherit;">此外，</font></font><code>max-height</code><font style="vertical-align: inherit;"><font style="vertical-align: inherit;">基于解决方案</font><font style="vertical-align: inherit;">的宽松程度和时机被打破</font><font style="vertical-align: inherit;">，因为使用的高度可能比达到高度</font></font><code>max-height</code><font style="vertical-align: inherit;"><font style="vertical-align: inherit;">要快得多</font></font><code>9999px</code><font style="vertical-align: inherit;"><font style="vertical-align: inherit;">。</font><font style="vertical-align: inherit;">随着屏幕分辨率的提高，象素的长度会</font></font><code>9999px</code><font style="vertical-align: inherit;"><font style="vertical-align: inherit;">在我的嘴里留下不良的味道。</font><font style="vertical-align: inherit;">我认为，这种特殊的解决方案以一种优雅的方式解决了这个问题。</font></font></p>

<p><font style="vertical-align: inherit;"><font style="vertical-align: inherit;">最后，这里希望CSS的未来修订版能够满足作者的要求，使他们更加优雅地做这些事情-重新审视“ compute” vs“ used”和“ resolved”值的概念，并考虑是否应将过渡应用于计算值。值，包括使用</font></font><code>width</code><font style="vertical-align: inherit;"><font style="vertical-align: inherit;">和</font><font style="vertical-align: inherit;">转换</font></font><code>height</code><font style="vertical-align: inherit;"><font style="vertical-align: inherit;">（目前有一些特殊处理）。</font></font></p></div>
